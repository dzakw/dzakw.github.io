Section 1: HTML

HTML Formatting

  • HTML Style Guide

  • Always declare the document type with <!DOCTYPE html> as the first line in a HTML documents
  • Always use lowercase for HTML element names
    Example:
    Good one: 
    <body>
    This is a code and content inside the <body> tag
    .....
</body>
    Bad one: 
    <BODY>
    This is a code and content inside the <BODY> tag
    .....
</BODY>
  • Always close HTML Elements (The code usually still works, but it is highly recomended to always close HTML elements
    Example:
    Good one: 
    <p>This is a paragraph</p>
    Bad one: 
    <p>This is a paragraph
  • Always use lowercase for attribute names
    Example:
    Good one: 
    <img src="image.jpg" alt="Image">
    Bad one: 
    <img src="image.jpg" ALT="Image">
  • Always use double quote for attribute values Example:
    Good one: 
    <img src="image.jpg" alt="Image">
    Bad one: 
    <img src=image.jpg alt=Image>
  • Always specify alt, width, and height for images. Example:
    Good one: 
    <img src="image.jpg" alt="Image" width="500" height="auto">
    Bad one: 
    <img src="image.jpg">
  • no need to insert spaces around equal sign Example:
    Good one: 
    <img src="image.jpg" alt="Image" width="500" height="auto">
    Bad one: 
    <img src = "image.jpg" alt = "Image" width = "500" height = "auto">
  • Insert blank lines and indentations to make the code more readable Example:
    Good one: 
    <body>
    <h1>This is a heading</h1>
    <p>This is a paragraph</p>

    <h2>This is a sub-heading</h2>
    <p>This is a paragraph</p>

    <h2>This is another sub-heading</h2>
    <p>This is a paragraph</p>
</body>
    Bad one: 
    <body>
<h1>This is a heading</h1>
<p>This is a paragraph</p>
<h2>This is a sub-heading</h2>
<p>This is a paragraph</p>
<h2>This is another sub-heading</h2>
<p>This is a paragraph</p>
</body>
  • Always add <title> element, as it is important for SEO, it will shown when the website is bookmarked, and it is shown in the browser tab bar
  • <html>, <body>, <head>> can be omited (not included) in HTML5, but it is still recommended to always include them
  • In HTML, it is optional to close an empty elements. However, if you want to make your HTML code compatible with XHTML, you must include the closing slash (/) in empty elements like below:
    <meta charset="UTF-8">
<meta charset="UTF-8" />
  • To ensure proper interpretation and correct search engine indexing, both the language and the character encoding <meta charset="charset"> should be specified in the <head> section of the HTML document
  • Inline styles have the highest specificity, overriding any other styles defined in internal or external stylesheets.

  • A file path describes the location of a file in a web site's folder structure. It can be absolute or relative.
  • An absolute file path is the full URL to a file, like below example:
    <img src="https://www.w3schools.com/images/picture.jpg" alt="Tiger" style="width:500px;height:600px">
    • Above code will display an image from the URL https://www.w3schools.com/images/picture.jpg
  • A relative file path points to a file relative to the current page, like below example:
    <img src="media/img1.png" alt="Tiger" style="width:500px;height:600px">
    • Above code will display an image from the URL media/img1.png which is located in the same folder as the current page.
    Below is file path example:
    Path Description
    <img src="picture.jpg"> The "picture.jpg" file is located in the same folder as the current page
    <img src="images/picture.jpg"> The "picture.jpg" file is located in the images folder in the current folder
    <img src="/images/picture.jpg"> The "picture.jpg" file is located in the images folder at the root of the current web
    <img src="../picture.jpg"> The "picture.jpg" file is located in the folder one level up from the current folder
  • The best practice is to use relative file path, because it will make the website more portable and easier to maintain.

HTML Semantics

  • HTML <head> element

  • The <head> element is a container for metadata (data about data) and is placed between the <html> tag and the <body> tag.
  • Metadata is data about the HTML document. Metadata is not displayed, but will used by the browsers.
  • Metadata typically define the document title, character set, styles, scripts, and other meta information.
  • Below is some usage of the <meta> tag inside the <head> element:

    • Define the character set used:
      •                             
      <meta charset="UTF-8">
    • Define keywords for search engines:
      •                             
      <meta name="keywords" content="HTML, CSS, JavaScript">
    • Define a description of your web page:
      •                             
      <meta name="description" content="Learn Frontend">
    • Define the author of a page:
      •                             
      <meta name="author" content="Iqbal Dzakwan">
    • Refresh document every 30 seconds:
      •                             
      <meta http-equiv="refresh" content="30">
    • Set the viewport to make your website look good on all devices:
      •                             
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
    • The viewport is the user's visible area of a web page. It should be included in every web page.

  • <head> element is a container for following elements:
    • <title> - defines a title for the document. It will display the title in the browser's title bar
    • <base> - specifies the base URL/target for all relative URLs in a document
    • <link> - defines the relationship between a document and an external resource (most used to link to stylesheets)
    • <meta> - provides metadata about the HTML document
    • <style> - defines the style information for a document
    • <script> - defines client-side JavaScript (only put critical script here, for less critical script, put before </body>)
    • <noscript> - defines an alternate content for users that have disabled scripts in their browser

  • Forms also has some useful attributes. Below are the list of the attributes:
    • accept-charset
    • action
    • autocomplete
    • enctype
    • method
    • name
    • novalidate
    • rel
    • target

  • accept-charset

    Specifies the character encodings that are to be used for the form submission

                                    
    <form accept-charset="UTF-8">

  • action

    Specifies where to send the form-data when a form is submitted

                                    
    <form action="https://www.example.com">

  • autocomplete

    Specifies whether a form should have autocomplete on or off

                                    
    <form autocomplete="on">

  • enctype

    Specifies how the form-data should be encoded when submitting it to the server

                                    
    <form enctype="multipart/form-data">

  • method

    Specifies the HTTP method to use when sending form-data
    The method attribute can have one of the following values:

    • get - The form data is appended to the URL in name/value pairs
      • Appends the form data to the URL, in name/value pairs
      • NEVER use GET to send sensitive data! (the submitted form data is visible in the URL!)
      • The length of a URL is limited (2048 characters)
      • Useful for form submissions where a user wants to bookmark the result
      • GET is good for non-secure data, like query strings in Google

    • post - The form data is sent to the server as a block of data in the body of the form
      • Appends the form data inside the body of the HTTP request (the submitted form data is not shown in the URL)
      • POST has no size limitations, and can be used to send large amounts of data.
      • Form submissions with POST cannot be bookmarked
      • Always use post if the form data contains sensitive or personal information
                                    
    <form method="post">

  • name

    Specifies the name of the form

                                    
    <form name="form1">

  • novalidate

    Specifies that the form should not be validated when submitted

                                    
    <form novalidate>

  • rel

    Specifies the relationship between the current document and the linked document

                                    
    <form rel="nofollow">

  • target

    Specifies where to display the response that is received after submitting the form.
    The target attribute can have one of the following values:

    • _blank - The response is displayed in a new window or tab
    • _self - The response is displayed in the same frame (this is default)
    • _parent - The response is displayed in the parent frame
    • _top - The response is displayed in the full body of the window
    • framename - The response is displayed in a named frame
                                    
    <form target="_blank">
